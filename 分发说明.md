# 分发说明 —— 怎么把「数学建模助教」发出去

本文写给**你（维护者）**：如何构建安装包、发布到各下载渠道，以及怎么让别人找到并用上它。
面向最终用户的说明在 [使用指南.md](使用指南.md)。

---

## 一、产物长什么样

打包脚本会在 `packaging/dist/` 生成（每个平台一份）：

| 平台 | 图形安装程序 | 免安装压缩包 |
|------|-------------|-------------|
| Windows | `数学建模助教-安装程序-v1.0.0.exe` | `数学建模助教-windows-x64.zip` |
| macOS | —（用压缩包） | `数学建模助教-macos-arm64.tar.gz` |
| Linux | —（用压缩包） | `数学建模助教-linux-x86_64.tar.gz` |

每个包都**自带 Python 运行环境**（约 600–800 MB 解压后），用户无需自己装 Python。
之所以这么大：助教要在本机跑模型生成的 Python 代码、还带一个 IDE 终端，必须捆绑真实解释器和 numpy/pandas/scipy/matplotlib 等科学计算库。

## 二、本地构建

### Windows（你当前的机器）

```powershell
# 在 packaging/ 目录
powershell -ExecutionPolicy Bypass -File build_windows.ps1
```

- 自动下载内置 Python、装依赖、拷 app、编启动器、打 zip。
- 若本机装了 [Inno Setup 6](https://jrsoftware.org/isdl.php)，还会自动编出图形安装程序；没装就只出 zip（加 `-SkipInstaller` 可显式跳过）。

### macOS / Linux

必须**在对应系统上**构建（捆绑的是该平台的原生 Python 与启动器）：

```bash
cd packaging
chmod +x build_posix.sh
./build_posix.sh
```

没有 Mac/Linux 机器也没关系——交给下面的 CI 自动构建。

## 三、三平台自动构建（GitHub Actions）

仓库已带 `.github/workflows/release.yml`：**打一个版本 tag 就会三平台并行构建并自动发布到 Releases。**

```bash
git tag v1.0.0
git push origin v1.0.0
```

几分钟后，Releases 页就会出现 Windows / macOS / Linux 三套包。
也可在仓库 Actions 页手动触发（workflow_dispatch）只构建、不发布，用来验证流程。

> 想改内置 Python 版本：编辑 `release.yml` 顶部的 `PY_VERSION` / `PBS_TAG`，
> 或给本地脚本传 `-PyVersion` / `PY_VERSION=`。版本组合必须在
> [python-build-standalone releases](https://github.com/astral-sh/python-build-standalone/releases) 里存在。

## 四、把仓库推上 GitHub（首次）

> ⚠️ 你这台机器当前**没装 git**。先装 [Git for Windows](https://git-scm.com/download/win)，
> 然后在 `d:\revent` 目录执行：

```powershell
git init
git add .
git commit -m "初始提交：数学建模助教 + 计费网关 + 打包分发"
# 在 GitHub 上建一个空仓库后：
git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git branch -M main
git push -u origin main
```

根目录的 `.gitignore` 已排除 `.venv`、用户数据、密钥（`.env`）和打包中间产物，可放心提交。

发布版本：

```powershell
git tag v1.0.0
git push origin v1.0.0   # 触发 CI 自动出三平台包
```

## 五、各下载渠道怎么铺

### 1. GitHub Releases（主渠道，自动）
打 tag 后 CI 自动上传，用户访问
`https://github.com/<你>/<仓库>/releases/latest` 即可下载对应系统的包。这是最稳、可直链的渠道。

### 2. 云盘（国内用户友好）
GitHub 在国内下载偶尔慢。把 Releases 里的包再传一份到百度网盘 / 阿里云盘 / 夸克：
- 新建「数学建模助教」共享文件夹，按版本分子目录（`v1.0.0/`）。
- 把分享链接（+提取码）写进下方「对外文案」与项目主页。

### 3. 源码下载（开发者）
- Releases 自带「Source code (zip/tar.gz)」。
- 或让开发者直接 `git clone`，按 [agent/README.md](agent/README.md) 从源码跑。

### 4. （可选）第三方软件站
如需上架国内软件站，用 Windows 图形安装程序那个 exe 提交，附 [使用指南.md](使用指南.md) 作为说明。

## 六、上线前必做：配置「用我们提供的额度」这条路

新手向导里「用我们提供的额度」按钮，需要你先把计费网关（`gateway/`）部署到公网，
再把网关地址告诉 app。两种方式任选：

**A. 改默认值（推荐，让所有用户开箱即用）** —— 在打包前编辑 `agent/.env`（或在
`agent/backend/config.py` 里改默认值）：

```ini
GATEWAY_SIGNUP_URL=https://你的网关域名/        # 用户注册领 Key 的站点
GATEWAY_BASE_URL=https://你的网关域名/v1         # OpenAI 兼容端点
GATEWAY_MODEL=deepseek-chat
```

**B. 留空** —— 向导里「用我们提供的额度」会显示为「未开启」，用户只能走「用我自己的 API Key」。
这条路无需任何后端，零成本，适合先发出去试水。

网关本身的部署见 [gateway/README.md](gateway/README.md)（务必改 `AUTH_SECRET` / `ADMIN_TOKEN`、走 HTTPS）。
